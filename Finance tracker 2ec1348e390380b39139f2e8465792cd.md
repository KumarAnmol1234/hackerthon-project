This project is a **web/mobile application** designed to help students **track and manage their monthly expenses** in a simple and visual way.

The user starts by entering their **monthly income or principal amount**. Throughout the month, they can add their **expenses**, categorized into different types such as food, travel, education, entertainment, etc.

At the end of the month, the application automatically:

- Calculates **total expenditure**
- Calculates **remaining savings**
- Provides a **clear financial summary**

To make the data easy to understand, the app represents information using **interactive graphs**:

- A **pie chart** to show category-wise spending distribution
- A **bar or line graph** to compare income, expenses, and savings over time

All user inputs are **stored securely in a database**, allowing:

- Monthly history tracking
- Data reuse for analytics and visualization
- Consistent performance across sessions

The goal of the project is to help students **develop better financial awareness and saving habits** using data visualization and simple tracking.

## 🧩 **Verbal System Architecture – Student Expense Tracker**

### 🔹 1️⃣ User Layer (Frontend)

At the top level, we have the **user interface**.

- The user interacts with a **web or mobile application**
- They can:
    - Enter monthly income (principal amount)
    - Add daily expenses with categories
    - View savings and summaries
- The frontend also displays:
    - Pie charts for category-wise expenses
    - Bar/line graphs for income vs expenses vs savings

📌 This layer focuses only on **input and visualization**.

---

### 🔹 2️⃣ Application Layer (Backend / API)

The frontend communicates with the **backend server** through APIs.

- The backend handles:
    - User requests (add expense, fetch monthly data)
    - Business logic:
        - Total expenses calculation
        - Savings calculation
        - Monthly summaries
    - Data validation (no negative amounts, valid categories)

📌 This is the **brain of the system**.

---

### 🔹 3️⃣ Data Layer (Database)

The backend stores and retrieves data from a **database**.

- The database stores:
    - User details
    - Monthly income
    - Expense entries (amount, category, date)
- Data is structured so that:
    - Monthly history can be tracked
    - Analytics can be generated easily

📌 This layer ensures **data persistence and reliability**.

---

### 🔹 4️⃣ Analytics & Visualization Flow

When the user opens the dashboard:

1. Frontend sends a request to backend
2. Backend fetches raw data from database
3. Backend processes totals and summaries
4. Processed data is sent back to frontend
5. Frontend renders graphs using chart libraries
